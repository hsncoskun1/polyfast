## 3. Tasarim Kararlari

### PTB ≠ Market Price
- PTB = openPrice = event baslangic fiyati (Chainlink source)
- outcome price (0.55/0.45) FARKLI kavram
- market price / live price FARKLI kavram
- Bunlar KARISTIRILMADI

### PTB Source
- Kaynak: __NEXT_DATA__ icindeki crypto-prices query
- Key pattern: ["crypto-prices", "price", "BTC", startTime, "fiveminute", endTime]
- Data: {"openPrice": 66920.89, "closePrice": null}
- Canli dogrulamada sayfadaki "Price To Beat: $66,920.90" ile BIREBIR eslesti
- PUBLIC API ENDPOINT YOK — SSR parse zorunlu (bilinen kisit)
- Fallback: past-results query (son event'in closePrice'i — varsayim, garanti degil)

### Neden bu kaynak secildi
- Dogrudan public API endpoint bulunamadi (gamma-api, clob, data-api → 404)
- __NEXT_DATA__ icindeki crypto-prices query Polymarket'in kendi gosterdigi PTB ile birebir eslesiyor
- Chainlink dogrudan kullanmak → Polymarket ile eslesmeme riski (kullanici reddetti)

### Lock Semantigi
- PTB bir kez alindiktan sonra KILITLENIR
- Lock sonrasi overwrite YASAK (RuntimeError firlatir)
- Ayni event icinde PTB degismez

### Retry Davranisi
- retry_max = 3 (HARDCODED — ileride config'den gelecek, su an ayar UI yok)
- Retry yalnizca PTB WAITING durumundayken calisir
- PTB alindiktan sonra retry DURUR
- Max retry asildiginda FAILED status

### Discovery Kabul Notu
- Discovery mantigi kesinlesti: tek slot, 7 asset, look_ahead yok
- Formul: end_ts = ((now // 300) + 1) * 300
- GET /markets?slug={asset}-updown-5m-{end_ts}
- Bu surumde discovery redesign yapilmadi — sadece kabul notu
