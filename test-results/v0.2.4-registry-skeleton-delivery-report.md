## 3. Tasarim Kararlari

### EventStatus State Machine (7 durum)
- DISCOVERED → VALIDATING, CLOSED
- VALIDATING → ACTIVE, INACTIVE, CLOSED
- ACTIVE → INACTIVE, EXPIRED, SUSPENDED, CLOSED
- INACTIVE → ACTIVE, EXPIRED, CLOSED
- EXPIRED → CLOSED
- SUSPENDED → ACTIVE, CLOSED
- CLOSED → (terminal, gecis yok)

### RegistryRecord vs DiscoveredEvent AYRI
- DiscoveredEvent = ham candidate (frozen, discovery-phase)
- RegistryRecord = authoritative lifecycle state (mutable status, transition methods)
- RegistryRecord ek alanlar: event_id, status, first_seen_at, last_seen_at, status_changed_at, has_open_position
- DiscoveredEvent'te bunlar YOK

### EventRegistry Service
- register_candidate: DiscoveredEvent alir, RegistryRecord olusturur (DISCOVERED)
- Ayni condition_id tekrar gelirse duplicate olusturmaz, last_seen gunceller
- transition_event: gecis kurali kontrol eder, gecersizse InvalidTransitionError
- get_by_status, get_by_condition_id, get_all query methodlari

### Gecis Kontrolu
- ALLOWED_TRANSITIONS dict ile tanimli
- Her state icin izin verilen hedef state'ler acik
- CLOSED terminal — hicbir yere gidemez
- Gecersiz gecis InvalidTransitionError firlatir
- Her gecis status_changed_at gunceller
