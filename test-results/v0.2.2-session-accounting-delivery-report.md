## 3. Tasarim Kararlari

### SessionStatus Enum (belirsiz string degil)
```python
class SessionStatus(str, Enum):
    ACTIVE = "active"
    FAILED_START = "failed_start"
```
GPT talimati geregi belirsiz string kullanilmadi. Minimal ama kontrollü enum. Ileride CLOSING, CLOSED, RECOVERED eklenebilir.

### SessionAccounting vs BalanceSnapshot Ayrimi
- `BalanceSnapshot` = ham API fetch sonucu (frozen, immutable)
- `SessionAccounting` = muhasebe katmani kaydi (session_id, status, start state)
- Ayni model DEGILLER. Bilinçli ayrildi.

### Duplicate Session Davranisi
- Aktif session varken yeni session olusturma → `DuplicateSessionError` firlatilir
- REDDEDILIR, mevcut session donmez, yeni session uretilmez
- Caller once mevcut session'i sonlandirmali

### StartupGuard vs SessionManager Ayirimi
- StartupGuard: acilis gating, balance fetch, izin/blok karari
- SessionManager: session olusturma, persist, load ownership
- Guard icinde session domain mantigi YOK
- SessionManager icinde StartupGuard import'u YOK

### Persistence
- Migration 002: tek tablo (`session_accounting`), minimal alanlar
- `session_id TEXT PRIMARY KEY, data_json TEXT, created_at TEXT`
- JSON serialize/deserialize ile roundtrip
- Ileride genisleyecek diye gereksiz alan eklenmedi
