## 3. Tasarim Kararlari

### Soft-Remove Mantigi
- Event scan'de gorulmezse miss count artirilir
- DELIST_THRESHOLD (3) consecutive miss sonrasi INACTIVE'e gecirilir
- Hemen silinmez — gecici kayip ≠ kalici kapanis
- Miss count event tekrar gorulunce SIFIRLANIR

### Open Position Protection
- has_open_position=True olan eventler ASLA soft-remove edilmez
- 10+ consecutive miss olsa bile korunur
- Log WARNING uretilir

### Restoration
- INACTIVE event tekrar scan'de gorulurse ACTIVE'e restore edilir
- Miss count sifirlanir

### Terminal State Skipping
- CLOSED ve EXPIRED eventler sync tarafindan ATLANIR
- Durumu degistirilmez

### No Destructive Delete
- Registry count ASLA azalmaz
- Event silmek YOK — yalnizca state gecisleri

### State Machine Guncelleme
- DISCOVERED > INACTIVE gecisi ALLOWED_TRANSITIONS'a eklendi
- Gerekce: discovery'de bulunan ama sonraki scanlarda gorulmeyen event INACTIVE olabilir
- Bu gecis safe sync icin gerekli
