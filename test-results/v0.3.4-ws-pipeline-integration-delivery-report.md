## 3. Mimari Kararlar

### 3-Katmanli Ayirma (Transport → Bridge → Pipeline)
```
RTDSClient (transport)
  ↓ raw JSON callback
WSPriceBridge (routing)
  ↓ token_id → (condition_id, asset, side)
LivePricePipeline (normalization)
  → LivePriceRecord (authoritative state)
```
- RTDSClient pipeline'i BILMEZ (import yok)
- WSPriceBridge her iki tarafi birlestirir
- Pipeline hangi kaynaktan geldigini source field ile izler

### WS Kaynak Onceligi
- WS = primary authoritative source (PriceSource.RTDS_WS)
- Gamma = bootstrap + fallback (PriceSource.GAMMA_OUTCOME_PRICES)
- Her record'un source field'i hangi kaynaktan geldigini gosterir
- WS → Gamma gecisi STALE durumunda otomatik mumkun

### Fiyat Hesaplama (WS)
- UP token: up_price = best_bid, down_price = 1 - best_bid
- DOWN token: down_price = best_bid, up_price = 1 - best_bid
- Spread = best_ask - best_bid (WS'ten, Gamma'dan DEGIL)
- best_bid/best_ask record uzerinde saklanir

### Auto-Reconnect + Resubscribe
- run_forever(): connect → subscribe → receive → reconnect dongusu
- Reconnect: exponential backoff, max 10 deneme, max 60s bekleme
- Her reconnect sonrasi _subscribed_tokens listesinden otomatik resubscribe
- shutdown_event ile temiz kapatma

### Invalid WS Veri Filtreleme
- bid=0, ask=0, negatif, 1+ → INVALID (evaluation'a ULASMAZ)
- Tek bad tick mevcut valid datanin ustune YAZILMAZ
- WAITING state'teki record bad tick alirsa INVALID olur

### Stale Handling
- WS baglantisi duserse → record'lar threshold sonrasi STALE olur
- STALE durumda Gamma fallback guncelleyebilir → FRESH
- WS reconnect sonrasi yeni data → STALE'den FRESH'e donus
