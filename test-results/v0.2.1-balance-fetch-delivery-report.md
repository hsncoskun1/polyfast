## 3. Tasarim Kararlari

### StartupGuard Pattern
- StartupGuard bagimsiz servis olarak tasarlandi
- Trading engine henuz yok, bu yuzden lifespan wiring yapilmadi
- Guard sonucu StartupResult dataclass'inda tutuluyor: status, balance, trading_allowed, incidents
- Guard calistirilmadan trading_allowed = False (guvenli varsayilan)

### Balance Fetch
- `AuthenticatedTradingClient.fetch_balance()` → `GET /balance`
- API response `{"balance": "247.85", "available": "200.00"}` bekleniyor
- BalanceSnapshot frozen dataclass — degistirilemez, from_api_response factory metodu

### Startup Guard Flow
```
fetch_balance() basarili → StartupStatus.PASSED, trading_allowed=True
fetch_balance() basarisiz → StartupStatus.FAILED, trading_allowed=False
  → HealthIncident(severity=CRITICAL, category="accounting")
  → Log: balance_fetch_failure
  → Log: trading_engine_blocked
  → suggested_action: credential/connectivity kontrolu
```

### Credential Yoksa
- has_credentials False → ClientError(AUTH) → guard FAILED
- Sessiz gecis YOK
