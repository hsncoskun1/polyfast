## Scope

Frontend dashboard (Faz 8) için gerekli tüm backend endpoint kontratlarını
extend eder. Response shape'leri frontend `eventTypes.ts` aynasıyla uyumlu,
tüm yeni alanlar **optional / placeholder-first**: orchestrator runtime
henüz her alanı sürmüyorsa `None` döner, frontend null-safe tüketir.

ancak bazı runtime wiring'ler sonraki sürümlerde gelecek:

1. **Contract merge semantiği**

   - Fallback registry (`_COIN_METADATA_FALLBACK`) ile merge edilir
   - `orchestrator.get_coin_metadata()` gerçek implementasyonu
     **merge blocker değil** — fallback zaten çalışıyor

## Davranış Kararları

- **Enum align** (claims): backend internal `pending/success/failed`
  → frontend `RETRY/OK/FAIL`. Bilinmeyen değer → `RETRY` fallback
  **+ WARNING log** (enum drift operator'a görünür).
- **Rule state coerce**: bilinmeyen → `waiting` + WARN log.
- **Unknown idle_kind** → `no_events` + WARN log.
- **Search `signal_ready`**:
  - Raw `signal_ready` alanı varsa öncelikli
  - Yoksa `considered = rules - disabled` üzerinden türetilir
  - `disabled` kurallar signal hesabına dahil edilmez
  - Boş `considered` → `false`
- **External provider exception'ları** (search/idle): boş liste +
  WARN log. Sessiz bypass **YOK** — her failure iz bırakır
  (CLAUDE.md §4 external connectivity failure policy).
- **Coins metadata**: orchestrator provider > static fallback > boş.
  Provider exception → WARN + fallback (sessiz bypass değil, iz var).
